SETUP (2 minutes)
─────────────────────────────────────────────────────────────────────────────

  □ Navigate to ai-pipeline directory
    $ cd ai-pipeline

  □ Copy environment example
    $ cp .env.example .env

  □ Edit .env and add API keys:
    GEMINI_API_KEY=sk-...         (from aistudio.google.com/app/apikey)
    TAVILY_API_KEY=tvly-...       (from tavily.com)
    MISTRAL_API_KEY=...           (optional, from console.mistral.ai)

  □ Install dependencies
    $ npm install

  □ Verify TypeScript compiles
    $ npm run typecheck


OPTION 1: DIRECT TERMINAL TEST (Recommended) ⭐
─────────────────────────────────────────────────────────────────────────────

  □ Run the test script
    $ npm run test-pipeline

  □ Watch the progress bar move to 100%

  □ Verify output:
    ✓ See "[100%] ████████████████████ stage5_assembler"
    ✓ See "✅ SUCCESS — Course generated and validated"
    ✓ See "📚 COURSE SUMMARY" with title, modules, lessons
    ✓ See "📁 Full course JSON written to: test-output.json"

  □ Verify output file was created
    $ ls -lh test-output.json

  □ Inspect the generated course
    $ cat test-output.json | jq '.' | less

  □ Check specific fields
    $ jq '.course.title' test-output.json
    $ jq '.course.modules | length' test-output.json
    $ jq '.course.capstone.techStack' test-output.json

  RESULT: ✅ DIRECT TEST PASSED

OUTPUT VALIDATION
─────────────────────────────────────────────────────────────────────────────

              learningOutcomes, modules, subtitle, tags, title

  □ Module count check:
    $ jq '.course.modules | length' test-output.json
    Expected: 3-8 modules (depends on learningDuration)

  □ Lesson content check:
    $ jq '.course.modules[0].lessons[0].content | keys' test-output.json
    Expected: body, codeSnippets, commonPitfalls, estimatedReadMinutes,
              keyTakeaways, realWorldExample

  □ Body length check (minimum 300 characters):
    $ jq '.course.modules[0].lessons[0].content.body | length' test-output.json
    Expected: >= 300

  □ Quiz check (exactly 4 questions per module):
    $ jq '.course.modules[0].quiz.questions | length' test-output.json
    Expected: 4

  □ Capstone phases check (3-6 phases):
    $ jq '.course.capstone.phases | length' test-output.json
    Expected: 3-6

  □ Tech stack validation (uses real tools, not hallucinated):
    $ jq '.course.capstone.techStack' test-output.json
    Expected: Real tools matching domain (e.g., FastAPI, SQLAlchemy, PostgreSQL)

  □ Metadata check:
    $ jq '.meta | keys' test-output.json
    Expected: generatedAt, generationDurationMs, models, schemaVersion, tavilySources

  □ Schema version validation:
    $ jq '.meta.schemaVersion' test-output.json
    Expected: "1.0.0"
